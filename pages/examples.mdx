# Examples

### `SimpleModelViewer`

The lightweight inspection experience for quick model review:

- Upload-first workflow
- Scene objects panel
- Environment and lighting controls
- Click-to-select focus flow

### `Guided Tour`

A guided walkthrough demo that uses scene annotations as tour stops instead of
per-object bindings:

- SceneAnnotationMarker stops
- Built-in annotation navigation
- Camera flies to saved world positions
- Position-only markers, no object binding needed

[Live Demo](https://react-immersive.liveroom.dev/examples/guided-tour) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/guided-tour-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/guided-tour-demo)

### `Shareable View / Deep Link`

Create a client-ready link to an exact walkthrough state:

- Pick an object to fly the camera to it
- Hide selected objects from the walkthrough
- Copy or open a link that restores camera, selection, and visibility state
- Built with `useShareableViewerState`

[Live Demo](https://react-immersive.liveroom.dev/examples/shareable-view) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/shareable-view-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/shareable-view-demo)

### `AR Product Viewer`

A standalone, mobile-first retail demo for placing a model in the customer&apos;s
space:

- WebXR via `enableXR` on supported Android devices
- iOS Quick Look using `usdzUrl`
- Real-world initial scale using `arScaleMode="real-world"`
- Desktop-to-phone QR handoff for continuing the same experience in mobile AR
- Minimal viewer UI focused on the product and AR handoff

[Live Demo](https://react-immersive.liveroom.dev/examples/ar-product-viewer) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/ar-product-viewer-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/ar-product-viewer-demo)

### `Switch Demo`

A focused room-level example where two switches control four lights through app
state:

- Start with exported `objectBindings`.
- Add actions to the switch records.
- Each switch action includes `effects` that patch the relevant light bindings.
- Pass the bindings into `ModelViewer` as React state.
- Update the same bindings when `onAction` fires.
- Use `ModelViewer`'s custom `lights` prop to make the whole scene darker or brighter based on the bound light state.

[Live Demo](https://react-immersive.liveroom.dev/examples/switch-demo) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/switch-room-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/switch-room-demo)

### `Vehicle Configurator`

A product-configurator example that shows how BindingBuilder-exported object
bindings can drive grouped customization flows:

- Grouped body paint, wheel, and trim swatches patch multiple meshes at once
- Built-in `change-color` / `change-material` / `toggle-visibility` actions are
  reinterpreted as real configurator controls
- An aero-kit toggle hides or restores a whole package of parts
- Window tint and headlight state update the same binding map the viewer renders
- Clickable parts and side-panel controls stay in sync through React state

[Live Demo](https://react-immersive.liveroom.dev/examples/vehicle-configurator) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/vehicle-configurator-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/vehicle-configurator-demo)

### `Exploded View`

An assembly-inspection demo that separates a multi-part model into its
components with a single slider:

- `showExplodeControls` renders the built-in explode slider in the viewer
- Every bound mesh slides outward from the model center along its own direction
- Nested parts ride their exploding parent instead of being displaced twice
- Fully reversible — dragging back to zero reassembles the model exactly
- Best suited to static product / CAD models (engines, gadgets, vehicles)

[Live Demo](https://react-immersive.liveroom.dev/examples/exploded-view) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/exploded-view-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/exploded-view-demo)

### `Cinematic Tour`

An automatic camera that glides through a space on its own, like a film:

- The `cinematic` prop takes a list of `waypoints` (camera position + look-at
  keyframes)
- The camera follows a smooth Catmull-Rom spline through them at a constant
  on-screen speed, looping seamlessly
- `autoPlay` starts the glide once the model has loaded and framed
- A play / pause control appears in the viewer; grabbing the camera pauses it
- Waypoints are authored visually in BindingBuilder's Cinematic tab and carried
  on `sceneConfig.cinematic`

[Live Demo](https://react-immersive.liveroom.dev/examples/cinematic-tour) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/cinematic-tour-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/cinematic-tour-demo)

### `Animation Studio`

A playback studio for a rigged, animated GLB, driven entirely by
`useViewerAnimations`:

- Clip picker for every animation baked into the model
- Play, pause, and stop transport controls
- A scrubbable timeline bound to the live playback position
- Playback speed presets
- A loop-mode toggle (repeat vs. play once) driven through `sceneConfig.animations`

[Live Demo](https://react-immersive.liveroom.dev/examples/animation-studio) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/animation-studio-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/animation-studio-demo)

### `Texture Upload / Custom Print`

A custom-apparel demo where shoppers upload their own artwork onto a product:

- The built-in Upload Artwork action hands the file to `onTextureUpload`
- A real app uploads it to storage and returns a durable URL (here a data URL)
- The returned URL is committed to `style.material.texture.path`
- The print is broadcast across every mesh of the garment
- Shirt color presets drive the same binding map

[Live Demo](https://react-immersive.liveroom.dev/examples/texture-upload) · [Starter App](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/texture-upload-demo) · [Open in CodeSandbox](https://codesandbox.io/s/github/liveroom-technologies/react-immersive-docs/tree/main/examples/texture-upload-demo)

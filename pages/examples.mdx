# Examples

## Hosted Demos

### `SimpleModelViewer`

The lightweight inspection experience for quick model review:

- Upload-first workflow
- Scene objects panel
- Environment and lighting controls
- Click-to-select focus flow

Live route:
[`/examples/simple-model-viewer`](/examples/simple-model-viewer)

Starter app:
[`examples/simple-model-viewer-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/simple-model-viewer-demo)

### `ModelViewer`

The fuller `ModelViewer` experience used for the richer hosted demo:

- Object bindings and per-object actions
- Scene config wiring
- Measurement tools
- Annotation navigation
- XR flags and performance tuning

Live route:
[`/examples/model-viewer`](/examples/model-viewer)

Starter app:
[`examples/model-viewer-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/model-viewer-demo)

### `Guided Tour`

A guided walkthrough demo that uses scene annotations as tour stops instead of
per-object bindings:

- SceneAnnotationMarker stops
- Built-in annotation navigation
- Camera flies to saved world positions
- Position-only markers, no object binding needed

Live route:
[`/examples/guided-tour`](/examples/guided-tour)

Starter app:
[`examples/guided-tour-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/guided-tour-demo)

### `Shareable View / Deep Link`

Create a client-ready link to an exact walkthrough state:

- Pick an object to fly the camera to it
- Hide selected objects from the walkthrough
- Copy or open a link that restores camera, selection, and visibility state
- Built with `useShareableViewerState`

Live route:
[`/examples/shareable-view`](https://react-immersive.liveroom.dev/examples/shareable-view)

Starter app:
[`examples/shareable-view-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/shareable-view-demo)

A standalone, mobile-first retail demo for placing a model in the customer&apos;s
space:

- WebXR via `enableXR` on supported Android devices
- iOS Quick Look using `usdzUrl`
- Minimal viewer UI focused on the product and AR handoff

Live route:
[`/examples/ar-product-viewer`](https://react-immersive.liveroom.dev/examples/ar-product-viewer)

Starter app:
[`examples/ar-product-viewer-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/ar-product-viewer-demo)

### `Switch Demo`

A focused room-level example where two switches control four lights through app
state:

- Start with exported `objectBindings`.
- Add actions to the switch records.
- Each switch action includes `effects` that patch the relevant light bindings.
- Pass the bindings into `ModelViewer` as React state.
- Update the same bindings when `onAction` fires.
- Use `ModelViewer`'s custom `lights` prop to make the whole scene darker or brighter based on the bound light state.

Live route:
[`/examples/switch-demo`](/examples/switch-demo)

Starter app:
[`examples/switch-room-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/switch-room-demo)

### `Vehicle Configurator`

A product-configurator example that shows how BindingBuilder-exported object
bindings can drive grouped customization flows:

- Grouped body paint, wheel, and trim swatches patch multiple meshes at once
- Built-in `change-color` / `change-material` / `toggle-visibility` actions are
  reinterpreted as real configurator controls
- An aero-kit toggle hides or restores a whole package of parts
- Window tint and headlight state update the same binding map the viewer renders
- Clickable parts and side-panel controls stay in sync through React state

Live route:
[`/examples/vehicle-configurator`](/examples/vehicle-configurator)

Starter app:
[`examples/vehicle-configurator-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/vehicle-configurator-demo)

### `Animation Studio`

A playback studio for a rigged, animated GLB, driven entirely by
`useViewerAnimations`:

- Clip picker for every animation baked into the model
- Play, pause, and stop transport controls
- A scrubbable timeline bound to the live playback position
- Playback speed presets
- A loop-mode toggle (repeat vs. play once) driven through `sceneConfig.animations`

Live route:
[`/examples/animation-studio`](/examples/animation-studio)

Starter app:
[`examples/animation-studio-demo`](https://github.com/liveroom-technologies/react-immersive-docs/tree/main/examples/animation-studio-demo)
